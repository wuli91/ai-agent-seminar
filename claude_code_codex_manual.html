0. 가장 큰 철학 차이 — 이것만 알면 절반

Claude Code: "권한 규칙" 중심

OS 샌드박스 없이 시작 — 대신 도구 호출 단위 승인 + permissions.allow/deny 규칙 + 5단계 권한 모드(Shift+Tab 순환: default→acceptEdits→plan…)로 통제. 파일은 실제 파일시스템을 직접 수정하고 체크포인트(Esc×2 rewind)로 되돌립니다.

Codex: "OS 샌드박스 + 승인 정책" 조합

명령 실행을 OS 레벨 샌드박스(read-only / workspace-write / danger-full-access)에 가두고, 별도 축으로 승인 정책(untrusted / on-request / never)을 둡니다. 두 축의 조합이 Claude Code의 권한 모드 하나에 대응.

전환 멘탈 모델Claude Code의 "이 도구 호출을 허용할까?"라는 질문이, Codex에서는 "이 명령이 어느 샌드박스에서 돌고, 언제 나에게 물어볼까?"로 두 축으로 분해됩니다. 반대로 Codex 사용자는 Claude Code에서 샌드박스 대신 규칙(allow/deny)과 모드로 같은 통제를 한다고 생각하면 됩니다.

5. 권한 · 샌드박스 — 가장 헷갈리는 부분

두 도구의 통제 축이 다릅니다. Claude Code = 모드 1축, Codex = 샌드박스 × 승인 2축.

7. 설정 파일 구조

Claude Code — settings.json (5계층 병합)

우선순위: Managed > CLI 인자 > .claude/settings.local.json > .claude/settings.json > ~/.claude/settings.json

주요 키: model · defaultMode · permissions.allow/deny · env · hooks · fallbackModel · additionalDirectories

Codex — config.toml (단일 + 프로파일)

기본 ~/.codex/config.toml 하나 + -p 프로파일로 ~/.codex/<이름>.config.toml 레이어. 일회성 오버라이드는 -c key.path=value (TOML 파싱).

주요 키: model · model_reasoning_effort · approval_policy · sandbox_mode · [mcp_servers] · [features] · [projects."경로"]

전환 포인트: Claude Code의 "프로젝트 설정을 git에 커밋해 팀 공유" 패턴이 Codex에는 약함(설정이 ~/.codex 중심) — Codex에서 팀 표준은 AGENTS.md+플러그인으로 공유하는 편.

15. 전환 시 헷갈림 주의 목록

같은 이름, 다른 것

  • -p — Claude Code에선 print(헤드리스), Codex에선 profile(설정 레이어). Codex 헤드리스는 exec.
  • -c — Claude Code에선 continue(세션 계속), Codex에선 config 오버라이드(-c model="...").
  • 리뷰 — Claude는 스킬(/code-review), Codex는 서브커맨드(codex review)이자 TUI 커맨드.

다른 이름, 같은 것

  • CLAUDE.md ↔ AGENTS.md · settings.json ↔ config.toml · /permissions ↔ /approvals · claude -p ↔ codex exec · /branch ↔ codex fork · Auto Memory ↔ memories

한쪽에만 있는 것

Claude Code 고유

체크포인트/rewind (Esc×2) · Plan Mode(계획서 검토 후 실행) · 도구 단위 permissions 규칙 문법 · settings 5계층(팀 공유 커밋) · Workflows(멀티에이전트 오케스트레이션) · Cowork(비개발 데스크톱 에이전트) · git worktree 내장(--worktree)

Codex 고유

OS 샌드박스 3종(+codex sandbox 단독 실행) · --oss 로컬 모델(ollama/lmstudio) 내장 · codex mcp-server(자신을 MCP 서버로) · codex apply(클라우드 diff 로컬 적용) · --profile 설정 레이어 · codex features(기능 플래그 직접 토글)
