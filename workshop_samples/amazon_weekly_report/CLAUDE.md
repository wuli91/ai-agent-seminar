# Amazon 주간 리포트 자동화 규칙

## 데이터 위치
- 이번 주 CSV: `data/this_week/`  / 전주 CSV: `data/last_week/`
- 폴더마다 리포트 3종이 들어 있습니다 (Seller Central에서 받은 원본 형식 그대로):
  - `business_report.csv` — 매출·트래픽 (SKU별 세션·페이지뷰·바이박스·주문수량·전환율·매출액)
  - `advertising_report.csv` — 스폰서 광고 (SKU별 노출·클릭·광고비·광고매출·ACoS)
  - `returns_report.csv` — FBA 반품 **건별(1행=1건)** 원본. SKU별로 집계해서 반품 수를 구할 것.
- CSV마다 컬럼명·구분자·인코딩이 다를 수 있으니 **자동 감지**해서 읽고, SKU 키로 3종을 합칠 것.
  (예: 매출은 `SKU`, 광고는 `Advertised SKU` 컬럼 → 같은 SKU로 매핑)
- 금액 문자열의 `$`·천단위 콤마·`%` 기호는 숫자로 정제해서 계산.

## 계산 지표 (SKU별)
- 매출, 주문수량, 광고비, 광고매출, **ACoS(=광고비 ÷ 광고매출)**
- 전환율(business_report 값 사용), **반품률(=반품건수 ÷ 주문수량)**
- 모든 지표의 **전주 대비 증감(절대치 + %)** — 매출·주문은 ↑가 좋음, ACoS·반품률은 ↑가 나쁨

## 산출물
- `reports/` 폴더에 `weekly_YYYY-MM-DD.html` 생성 (YYYY-MM-DD = 이번 주 월요일)
- 구성:
  1. 요약 — 총매출·전주대비·블렌디드 ACoS·총 반품률 + **핵심 변화 3줄**
  2. SKU별 표 — 전주 대비 ▲▼ 표기
  3. 차트 — 매출 비중·ACoS 상위 (외부 라이브러리 없이 HTML/CSS 막대)
  4. **주의 신호** — 반품 급증·ACoS 급등·바이박스 하락·재고 소진 등 자동 코멘트
- 한국어 리포트. 숫자는 천단위 콤마, 통화 `$` 표기. 인쇄(A4) 친화적으로.

## 원칙
- CSV에 없는 값은 추측하지 말고 "데이터 없음"으로 표기.
- 계산식·데이터 출처는 리포트 하단에 명시.
- ⚠️ 이 폴더의 데이터는 **세미나용 가짜 데이터**입니다. 실제 판매 데이터로 바꿔 쓸 때는
  절대 공개 저장소(git)에 커밋하지 말 것.
