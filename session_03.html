핸즈온: 샘플 폴더로 실제 자동화 1건

왜 Claude Code인가 — 앱과의 차이

Claude 앱Claude Code
다루는 대상붙여넣은 글, 업로드한 몇 개 파일내 컴퓨터의 폴더·수십 개 파일
할 수 있는 일글쓰기·요약·분석파일 읽기·만들기·이름변경·변환·정리
반복매번 직접한 번 시키면 전부 자동
데이터업로드 필요내 컴퓨터 안에서 처리(파일 자체는 로컬)

"여러 파일을 손으로 반복"하던 일이 한 문장 지시로 끝납니다.

터미널, 사실 이것만 알면 됩니다

① 연다

맥: "터미널" · 윈도우: "PowerShell"

② claude 입력

그러면 AI와 대화 시작

③ 한국어로 지시

"이 폴더 정리해줘"처럼 말하면 끝

안심하세요명령어를 외울 필요 없습니다. 한국어로 부탁하면 Claude가 알아서 합니다. 우리는 승인만 하면 됩니다.

첫 실행 — 3단계 Claude Code

  1. 작업할 폴더를 하나 정한다 (예: 바탕화면의 실습 폴더)
  2. 터미널에서 그 폴더로 이동 후 claude 입력
  3. 한국어로 부탁 → Claude가 할 일을 보여주면 확인(승인)
# 맥: 폴더를 터미널로 끌어다 놓는 법이 가장 쉬움
cd ~/Desktop/실습
claude

# 그다음 그냥 한국어로:
이 폴더에 어떤 파일들이 있는지 정리해서 알려줘

⚠ 안전 장치 — 승인 없이는 안 바뀜

읽기는 자유

파일을 읽고 분석하는 건 그냥 진행됩니다.

바꾸기는 승인 후

파일을 만들거나 수정·삭제하기 전에 Claude가 먼저 보여주고 묻습니다. 내가 "예" 해야 실행.

3가지 습관 ① 중요 폴더는 복사본으로 실습 ② 승인 화면을 읽고 허락 ③ 의심되면 거절(No) 후 다시 지시.

비개발자도 바로 쓰는 활용

파일·폴더 정리

"날짜별 폴더로 분류해줘 / 중복 파일 찾아줘"

대량 이름·형식 변경

"사진 이름을 날짜_순번으로 / 이미지를 PDF로 합쳐줘"

데이터 정리

"이 CSV들에서 필요한 열만 뽑아 하나로 합쳐줘"

문서 일괄 생성

"이 명단으로 안내문 50개를 이름만 바꿔 만들어줘"

자료조사 정리

"이 텍스트 파일들을 읽고 주제별 요약표로"

회의록·메모 통합

"이 폴더 회의록에서 결정사항만 한 표로"

데모 — 흩어진 파일 한 번에 정리

(실습 폴더에서 claude 실행 후, 한국어로)

이 폴더의 파일들을 확장자별 폴더로 분류해줘.
- 이미지 / 문서 / 표(엑셀·csv) / 기타
- 옮기기 전에 어떤 식으로 옮길지 먼저 보여줘
- 원본은 지우지 말고 복사로 정리해줘

Claude가 계획을 보여줌 → 내가 승인 → 정리 완료. 손으로 30분 → 30초.

🙌 핸즈온 — 내 자동화 1건 Claude Code

  1. 샘플 폴더 workshop_samples를 연다 — 받는 법·내용물은 실습 소재 모음 ④ (또는 내 폴더의 복사본)
  2. 그 폴더에서 claude 실행
  3. "이 폴더에 뭐가 있는지 알려줘"로 먼저 파악
  4. 지난 세션에 적어둔 "반복되는 파일 작업"을 한국어로 시켜본다
  5. 승인 화면을 읽고 진행 → 결과 확인
막히면에러가 나도 그 메시지를 그대로 Claude에게 "이거 무슨 뜻이야?"라고 물어보세요. 설치·로그인·터미널 문제는 트러블슈팅 가이드에 증상별 해결법이 있습니다. 시킬 일이 마땅치 않으면 샘플 폴더 안의 과제 문서(채용 스크리닝·매출 리포트 등 한국어판 포함)를 그대로 시켜보세요.

Claude Code 잘 쓰는 법

먼저 '보여줘'

"바꾸기 전에 계획을 보여줘" → 안전하게 검토 후 승인.

작게 시작

폴더 하나, 작업 하나. 되면 점점 키우기.

복사본으로 연습

원본은 보존. 결과 확인 후 적용.

대화로 고친다

"방금 건 이렇게 바꿔줘"로 이어서 다듬기.

데이터·보안 — Code에서 특히

  • 로컬 처리: 파일은 내 컴퓨터에 있지만, 처리를 위해 내용이 Claude로 전송됩니다 → 기밀 폴더 주의.
  • 비밀번호·키·인증서가 든 폴더에서는 실행하지 않기.
  • 고객정보·계약 등은 회사 정책 확인 후, 가능하면 익명화 사본으로.
  • 삭제·이동은 항상 승인 화면을 읽고 진행.
회사 정책 우선보안팀이 정한 "허용 도구·허용 데이터" 범위를 항상 먼저 따릅니다.
