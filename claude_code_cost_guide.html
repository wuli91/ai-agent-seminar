Claude Code 비용 최적화 실전 적용

"개발자당 월 $300"을 낮추는 레버를 지금 쓰는 Claude Code 환경에서 실제로 어떻게 적용하는가. 명령·설정·습관 단위로 정리했습니다.

먼저 — 무엇이 자동이고 무엇이 우리 몫인가 자동 프롬프트 캐싱은 Claude Code가 기본으로 적용합니다(켤 것 없음). 우리가 하는 건 캐시를 깨지 않는 습관입니다. 우리 설정 모델 라우팅 · 컨텍스트 관리 · 배치 파이프라인 · 측정은 우리가 정합니다.

1. 프롬프트 캐싱 자동 — "깨지 않게" 쓰는 법

작동 원리(요약)

동일한 앞부분(prefix)(시스템 프롬프트·도구 정의·CLAUDE.md·이미 읽은 파일·대화 누적)을 재사용하면 그 부분 입력 토큰이 약 90% 저렴(캐시 read)해집니다. 캐시는 기본 5분(옵션 1시간) 유지되고, prefix가 바뀌면 캐시가 깨져 다시 정가로 청구됩니다. Claude Code가 이걸 자동으로 합니다.

캐시 히트를 높이는 습관 (= 비용↓)

· 한 작업은 끊기지 말고 연속으로 — 5분 이상 멈추면 캐시 만료 → 다음 턴이 정가. 자리 비울 거면 굵직한 질문을 몰아서.

· CLAUDE.md·자주 읽는 파일을 안정적으로 유지 — 세션 중 자주 바꾸면 매번 캐시 무효화. (편집 잦은 파일을 prefix 앞쪽에 고정하지 말 것)

· 같은 컨텍스트 재사용 — 큰 파일을 대화 중간에 계속 새로 통째 주입하지 말고, 한 번 읽힌 걸 이어서 활용.

· /clear는 캐시도 리셋 — 무관한 작업으로 넘어갈 때만(같은 작업 중엔 남발 금지).

· 대형 코드베이스를 오래(5분 이상 간격으로) 다루면 1시간 캐시로 히트율↑: export ENABLE_PROMPT_CACHING_1H=1. 구독(Max/Team/Ent)에선 추가비 없음(플랜에 포함). API 종량은 캐시 write가 더 비싸지므로 장시간 재사용 워크로드에서만 유리.

2. 모델 라우팅 우리 설정 — 가장 큰 레버

Haiku는 Opus 대비 약 25배 저렴(입력 $1 vs $5 등), Sonnet은 Opus의 절반대. 단순 작업을 비싼 Opus로 돌리는 게 가장 흔한 낭비입니다.

방법적용
세션 중 전환/model 입력 → Sonnet/Haiku/Opus 선택. 어려운 설계·디버깅만 Opus, 기본은 Sonnet, 잡일·요약은 Haiku.
팀 기본 모델 고정환경변수 또는 settings로 기본을 Sonnet으로(불필요한 Opus 사용 차단).
배경작업 저가화요약 등 배경 호출에 쓰는 small/fast 모델을 Haiku로 지정.
서브에이전트조사·리뷰 서브에이전트에 저가 모델 지정.
# 세션에서
/model sonnet        # 기본 작업
/model haiku         # 단순/대량 작업

# 팀 기본값 고정 (셸 프로필 또는 CI)
export ANTHROPIC_MODEL="claude-sonnet-4-5"
export ANTHROPIC_SMALL_FAST_MODEL="claude-haiku-4-5"  # 배경작업용
주의정확한 모델 ID·가용 모델은 플랜/버전에 따라 다릅니다 — /model 목록과 공식 문서에서 현재 ID 확인 후 적용하세요.

3. 컨텍스트·반복 다이어트 우리 습관

· /clear — 무관한 작업으로 전환 시 누적 컨텍스트 비우기(매 턴 딸려가는 토큰 제거).

· /compact "지시" — 긴 세션을 요약 압축. "수정 파일 목록·테스트 명령은 보존" 같은 지시를 함께.

· Plan Mode — 코딩 전 계획을 세워 헛반복(잘못된 길 되돌리기)을 줄임. 다중 파일·불확실 작업일수록 효과.

· CLAUDE.md 간결하게 — 비대하면 매 요청마다 그 토큰을 싣는다. "이 줄 빼면 실수하나?"로 가지치기. 가끔 쓰는 지식은 Skills로 온디맨드.

· 범위 지정 읽기 — 거대한 파일을 통째 read 시키지 말고 필요한 부분/디렉터리로 한정.

· 서브에이전트로 조사 격리 — 테스트 실행·문서 조회·로그 처리 등 장황한 작업을 서브에이전트에 위임(출력은 요약만 메인에).

· 확장적 사고(thinking) 조절 — thinking 토큰은 출력 토큰으로 과금(기본 ON, 요청당 수만 토큰까지). 단순 작업은 /effort를 낮추거나 /config에서 thinking 끄기, 또는 export MAX_THINKING_TOKENS=8000.

· MCP 대신 CLI — gh·aws·gcloud 등 CLI가 MCP보다 컨텍스트 효율적. /mcp로 안 쓰는 서버 끄고, /context로 무엇이 컨텍스트를 먹는지 점검.

· Hooks로 전처리 — 1만 줄 로그를 통째로 읽히는 대신, PreToolUse 훅으로 grep ERROR만 남겨 토큰을 수만→수백으로.

· 구체적 프롬프트 — "코드베이스 개선" 같은 막연한 요청은 광범위 스캔 유발. "auth.ts의 login에 입력검증 추가"처럼 좁히면 파일 read 최소화. 길 잘못 들면 Esc로 즉시 중단·/rewind.

4. 배치(−50%) × 캐싱(−90%) — 대량 비대화형 작업

중요인터랙티브 Claude Code 세션에는 배치 할인이 자동 적용되지 않습니다. 배치(−50%)는 Message Batches API의 비동기 작업용입니다.

적용법

· 대량·반복 작업(전 파일 테스트 생성, 일괄 리팩터, 문서 자동화, 코드 주석)은 야간 배치 파이프라인으로 분리.

· 헤드리스 Claude Code를 스크립트로 파일별 루프(권한 스코핑 포함):

# 파일 목록을 만들어 파일별로 비대화형 실행
for f in $(cat files.txt); do
  claude -p "이 파일에 단위 테스트 추가, 운영코드 변경 금지: $f" \
    --allowedTools "Edit,Bash(npm test)" --output-format json
done

· 진짜 −50%가 필요한 정형 대량 작업은 Message Batches API로 작성(엔지니어링 파이프라인). 캐싱과 겹치면 표준가의 ~5%까지.

5. 지출 상한(캡) 걸기 관리자/개인

방법적용
워크스페이스 지출 한도
(API/Enterprise)		Console의 "Claude Code" 워크스페이스에 spend limit 설정 → 전사 Claude Code 총지출 상한. rate limit(TPM/RPM)도 걸어 다른 운영 워크로드 보호.
개인 월 한도
(Pro/Max)		/usage-credits로 월 사용크레딧 상한 설정(초과 시 CLI에서 상향/해제 프롬프트).
정액 구독 전환헤비 dev를 Max 20x($200) 또는 Team Premium 좌석으로 → 종량을 정액 상한으로 고정($300→$200). 단 5h 롤링 한도/throttle 감수, 워크로드가 한도에 맞아야.

관리자 권장 rate limit(공식): 사용자 5~20명이면 1인 100k~150k TPM, 20~50명이면 50k~75k TPM 등 — 조직이 커질수록 1인 할당↓. 동시 사용이 적기 때문.

6. 측정·가시화 선결 조건

"누가 / 어떤 모델 / 어떤 작업"이 비싼지 모르면 절감 우선순위를 못 정합니다. 먼저 계측하세요.

수준방법
개인 세션/usage — 세션 토큰·비용 + 플랜 한도 대비 사용량(스킬·서브에이전트·MCP별 분해, 24h/7d 전환). /context로 현재 컨텍스트 점유 확인. ※ 구독자는 비용 대신 사용량 바 표시
관리자(Console)Anthropic Console Usage 페이지 — 일별 비용·사용자 수·1인당 월 지출. 권위 있는 청구 기준.
팀/Enterprise관리자용 Claude Code Analytics 대시보드(사용자·팀별 활동·토큰).
상세 관측OpenTelemetry 내보내기 → Grafana/SigNoz/Datadog(토큰·비용·캐시효율·세션·1인별 분해). MDM 관리설정으로 전사 일괄 켜기.
⚠ Bedrock/Vertex/Foundry로 Claude를 돌리면Claude Code가 클라우드에서 비용 메트릭을 보내지 않습니다. 이 경우 LiteLLM(오픈소스 LLM 게이트웨이)으로 키별 지출을 추적하는 방식이 많이 쓰입니다(비공식·보안 자체검토 필요).
# Claude Code OpenTelemetry 활성화 (예시)
export CLAUDE_CODE_ENABLE_TELEMETRY=1
export OTEL_METRICS_EXPORTER=otlp
export OTEL_EXPORTER_OTLP_ENDPOINT="http://:4317"
# → dev별·모델별·작업별 토큰/비용을 대시보드로 추적

공식 Grafana "Claude Code Usage" 대시보드 등 기성 템플릿 활용 가능. 정확한 env 변수/메트릭명은 공식 monitoring 문서 확인.
