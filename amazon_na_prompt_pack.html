북미 Amazon 프롬프트 묶음

리스팅·콘텐츠 / 광고(PPC) / 리뷰분석 / 키워드·리서치 — Claude 앱(채팅·Projects)에 바로 붙여 쓰는 검증된 프롬프트 모음입니다.

사용법 4가지
  • [ ]·(... 붙여넣기) 자리표시자를 내 정보로 교체하세요.
  • 데이터 필요 표시 프롬프트는 실데이터를 직접 붙여넣어야 합니다. AI는 실검색량·가격·BSR을 모릅니다.
  • 매번 쓰는 작업은 Projects에 브랜드 가이드·금칙어·예시 리스팅을 올려두면 일관성이 올라갑니다.
  • 마지막 줄에 "확실하지 않으면 추측하지 말고 모른다고 해"를 붙이면 환각이 줄어듭니다.

1. 리스팅 · 콘텐츠

① 리스팅 풀세트 생성 데이터 불필요

너는 미국 아마존 리스팅 카피라이터야. 아래 제품 정보로 US 마켓 리스팅을 만들어줘.

[제품명/카테고리]: ...
[핵심 스펙·소재·사이즈]: ...
[차별점 3가지]: ...
[타깃 고객]: ...
[꼭 넣을 키워드]: ...

출력:
1) Title (200자 이내, 핵심 키워드 앞쪽)
2) Bullet points 5개 (각 베네핏 중심, 대문자 남발 금지)
3) Product Description (단락형)
4) A+ Content 섹션 3개 (헤드라인 + 본문)
- 톤: 신뢰감, 미국 원어민 자연스러움. FTC상 과장·의학적 단정 표현 회피.
- 금지표현: best, #1, guaranteed cure 같은 단정/허위 소지 표현 제외.

② 기존 리스팅 진단·최적화 데이터 필요

아래는 우리 현재 리스팅과 상위 경쟁사 리스팅이야.
우리 것을 진단하고 개선안을 제시해줘.

[우리 리스팅]: (제목/불릿/설명 붙여넣기)
[경쟁사 리스팅 2~3개]: (붙여넣기)

출력:
- 우리 리스팅의 약점 5가지 (구체적으로)
- 누락된 키워드/베네핏
- 개선된 Title·Bullet 제안 (before→after 표)
- 전환율 관점 우선순위 Top 3

③ 한→영 마케팅 현지화 데이터 불필요

아래 한국어 마케팅 문구를 미국 아마존용 영어로 현지화해줘.
직역하지 말고, 미국 소비자가 자연스럽게 받아들이는 표현·관용구로.
한국식 과장은 빼고, 베네핏 중심으로. 2가지 버전(차분/활기) 제시.

(한국어 문구 붙여넣기)

2. 광고 (PPC)

① 캠페인 구조 설계 데이터 불필요

신제품의 아마존 Sponsored Products 캠페인 구조를 설계해줘.

[제품/카테고리]: ...
[목표]: 초기 런칭(노출 확보) / 또는 수익화(ACoS 관리)
[예산(월)]: $...

출력:
- Auto / Manual(Broad·Phrase·Exact) 캠페인 분리안
- 광고그룹·키워드 그룹핑 전략
- 시작 입찰·예산 배분 가이드(원칙)
- 2주 후 점검 체크리스트(어떤 지표로 무엇을 조정)
주의: 실검색량은 모르니 전략·구조만. 실제 키워드 볼륨은 별도 툴로 확인하겠음.

② 광고 리포트 분석 → 낭비 키워드·네거티브 데이터 필요

아래는 지난 30일 Sponsored Products 검색어 리포트(CSV)야.
분석해서 액션을 뽑아줘.

(CSV 붙여넣기 — 또는 파일 업로드)

출력:
- 전환 0인데 클릭·비용 높은 검색어 → 네거티브 후보 목록
- ACoS 높은(>목표) 키워드 → 입찰 인하 후보
- 전환 좋은(ACoS 낮은) 키워드 → 입찰 상향/Exact 승격 후보
- 표로 정리하고, 예상 절감액 큰 순으로 우선순위
[목표 ACoS]: ...%

③ 광고 헤드라인·카피 다량 데이터 불필요

아래 제품으로 Sponsored Brands 헤드라인을 만들어줘.
- 50자 이내, 미국 톤, 베네핏 중심
- 8종 제시 후 가장 추천 1개 표시. 금지: 과장·단정 표현
(제품 정보 붙여넣기)

3. 리뷰 · VOC 분석

① 대량 리뷰 감정분석·토픽 데이터 필요

아래는 우리 제품 리뷰 모음이야. 분석해줘.

(리뷰 텍스트/CSV 붙여넣기 — 또는 파일 업로드)

출력:
- 긍정/부정/중립 비율
- 자주 언급되는 칭찬 토픽 Top 5 (빈도와 대표 인용)
- 자주 언급되는 불만 토픽 Top 5 (빈도와 대표 인용)
- 별점 하락의 핵심 원인 가설 3가지
- 리스팅/제품에 즉시 반영할 액션 5가지

② 부정 리뷰 → 개선 액션 매핑 데이터 필요

아래 별점 1~3 리뷰들을 보고, 각 불만을
[제품 개선] / [리스팅 보완(오해 해소)] / [CS 응대] 중 어디로 풀지 분류해줘.
표: 불만요지 | 빈도 | 분류 | 구체 조치 제안
(부정 리뷰 붙여넣기)

③ 경쟁사 리뷰로 차별화 포인트 데이터 필요

아래는 경쟁사 제품 리뷰야. 경쟁사의 약점(고객 불만)을 뽑고,
우리가 리스팅·제품에서 강조하면 좋을 차별화 포인트로 변환해줘.
(경쟁사 리뷰 붙여넣기)

4. 키워드 · 시장 리서치

① 시드 키워드 확장·검색의도 분류 데이터 불필요

아래 제품의 아마존 키워드 후보를 확장해줘.
[제품/용도]: ...
[시드 키워드]: ...

출력:
- 롱테일·연관 키워드 30개 (영어)
- 검색의도별 분류: 정보형 / 비교형 / 구매임박형
- 백엔드 search term 후보(중복·금지어 제외)
주의: 실검색량은 모르니 후보·분류만. 볼륨은 Helium10 등으로 검증하겠음.

② 경쟁사 비교표 데이터 필요

아래 경쟁 제품들의 리스팅 정보로 비교표를 만들어줘.
열: 제품 | 가격대 | 핵심 베네핏 | 약점(리뷰 기반) | 우리 대비 포지션
마지막에 우리 제품의 빈틈(기회) 3가지.
(경쟁 제품 정보·리뷰 붙여넣기)

③ 시장 동향 리서치 웹검색 환경 필요

(웹 검색이 켜진 Claude 또는 리서치 기능에서)
