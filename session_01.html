AI 에이전트

목표를 주면 스스로 단계를 나눠 도구를 쓰고(파일 읽기·검색·실행) 일을 처리합니다.

예) "이 폴더의 회의록 20개를 읽고 결정사항만 한 표로 정리해줘" → 에이전트가 파일을 직접 열어 읽고 → 정리 → 결과 파일 생성까지.

이번 세미나의 Claude 앱은 대화형, Claude Code는 내 컴퓨터에서 일하는 에이전트입니다.

좋은 프롬프트 6원칙

① 역할을 준다

"너는 10년차 마케터야."

② 맥락을 준다

"대상은 40대 임원, 톤은 정중하게."

③ 구체적 작업

"3문단으로, 제목 3개 후보와 함께."

④ 형식을 정한다

"표로 / 불릿으로 / 200자 이내로."

⑤ 예시를 보여준다

"이런 느낌으로: (좋은 예 붙여넣기)"

⑥ 반복해 다듬는다

"더 짧게 / 더 친근하게 / 숫자 추가."

같은 일, 프롬프트 차이

아쉬운 요청

회의록 요약해줘

→ 무난하지만 내가 원하는 형태가 아님

좋은 요청

아래 회의록을 정리해줘.
- 결정사항 / 담당자 / 기한을 표로
- 미해결 이슈는 따로 불릿으로
- 임원 보고용, 한 페이지 분량

(회의록 붙여넣기)

차이는 '무엇을, 누구에게, 어떤 형식으로'를 말해줬는지입니다.

초보가 자주 하는 실수

검증 없이 그대로 사용

숫자·사실·인용은 반드시 확인. AI는 틀려도 자신 있게 말합니다.

한 번에 포기

첫 답이 아쉬워도 정상. "다시, 이렇게"로 다듬으세요.

너무 막연한 요청

맥락·형식을 안 주면 평범한 답만 나옵니다.
