Claude Code 실습 (시나리오 A) · 핸즈온

Amazon 주간 성과 리포트 자동화

매주 손으로 만들던 매출·광고 리포트를 Claude Code로 자동화합니다. 누구나 따라 하는 레벨 1(CSV 기반)부터, 개발형 레벨 2(SP-API 연동)까지.

목표 산출물 SKU별 매출·ACoS·반품률을 계산하고, 전주 대비 변화 표 + 차트가 들어간 주간 리포트(PDF/HTML)를 자동 생성. 레벨 2에서는 매주 월요일 자동 실행까지.
이 실습의 현실적 설계 세미나에서 SP-API 전체 연동은 무겁습니다. 그래서 레벨 1 = Seller Central에서 수동 다운로드한 CSV로 먼저 자동화를 완성하고, 레벨 2 = 다운로드까지 API로 자동화 + 스케줄로 확장합니다. 레벨 1만으로도 "리포트 작성" 자체는 100% 자동화됩니다.

0. 사전 준비

  • Claude Code 설치 (세미나 3회차 사전 준비 — 가이드)
  • 실습 폴더 1개. Seller Central → Reports에서 지난주/전주 CSV를 받아 둡니다 (매출·트래픽, 광고 검색어 리포트 등).
  • 레벨 2만: SP-API/Amazon Ads API 개발자 등록·자격증명(LWA) — 개발 인력 동반 권장.

1. 레벨 1 — CSV 기반 자동 리포트 누구나

① 폴더 구조 만들기

amazon-weekly/
├─ data/
│  ├─ this_week/      ← 이번 주 CSV 넣기 (매출, 광고 등)
│  └─ last_week/      ← 전주 CSV 넣기
├─ reports/           ← 결과물이 여기에 쌓임
└─ CLAUDE.md          ← Claude Code에게 주는 규칙

② CLAUDE.md 작성 (Claude Code의 작업 규칙)

폴더에 CLAUDE.md를 만들어 두면 Claude Code가 매번 이 규칙을 읽고 일관되게 일합니다.

# Amazon 주간 리포트 자동화 규칙

## 데이터
- 이번 주 CSV: data/this_week/  / 전주 CSV: data/last_week/
- CSV 인코딩·구분자가 섞여 있을 수 있으니 자동 감지해서 읽을 것.

## 계산 지표 (SKU별)
- 매출, 주문수, 광고비, 광고매출, ACoS(=광고비/광고매출),
  전환율, 반품률, 전주 대비 증감(%)

## 산출물
- reports/ 폴더에 "weekly_YYYY-MM-DD.html" 생성
- 구성: ① 요약(총매출·총ACoS·주요 변화 3줄) ② SKU별 표(전주 대비 ▲▼)
  ③ 차트(매출 추이·ACoS 상위) ④ 주의 신호(재고소진·ACoS급등) 자동 코멘트
- 한국어 리포트. 숫자는 천단위 콤마, 통화 $ 표기.

## 원칙
- CSV에 없는 값은 추측하지 말고 "데이터 없음"으로 표기.
- 계산식은 리포트 하단에 명시.

③ Claude Code에 시키기 (자연어 한 줄)

data 폴더의 이번주/전주 CSV를 읽어서 CLAUDE.md 규칙대로
주간 리포트 HTML을 reports/에 만들어줘. 먼저 어떤 컬럼을 어떻게
매핑했는지 보여주고, 내가 확인하면 리포트를 생성해.

Claude Code가 CSV 구조를 파악 → 매핑 제안 → (승인) → 정제·계산·차트·HTML 생성까지 수행합니다. 결과가 어긋나면 그대로 대화로 수정 지시("ACoS 계산이 틀렸어, 광고비/광고매출이야").

레벨 1 완료 = 사람이 하던 '작성'은 끝 이제 매주 CSV만 폴더에 넣고 같은 한 줄을 치면 리포트가 나옵니다. 다운로드와 실행 클릭만 남았고, 그건 레벨 2에서 없앱니다.

2. 레벨 2 — API 연동 + 스케줄 개발형

① 자격증명 보관 (보안 필수)

amazon-weekly/
├─ .env            ← API 키/토큰 (절대 git에 올리지 않음)
├─ .gitignore      ← .env, data/ 제외
└─ scripts/        ← Claude Code가 만든 수집 스크립트
# .env  (예시 — 실제 값은 본인 것)
SP_API_REFRESH_TOKEN=...
SP_API_CLIENT_ID=...
SP_API_CLIENT_SECRET=...
ADS_API_REFRESH_TOKEN=...
ADS_API_PROFILE_ID=...

② Claude Code에 시키기

.env의 자격증명으로 Amazon SP-API에서 지난주 매출 리포트를,
Ads API에서 광고 리포트를 자동으로 받아 data/this_week/에 저장하는
스크립트를 scripts/에 만들어줘. 토큰은 코드에 박지 말고 .env에서 읽고,
요청 실패 시 재시도와 에러 로그를 넣어. 받은 다음 레벨1 리포트 생성까지
한 번에 실행되는 명령으로 묶어줘.

※ SP-API 리포트 종류·필요 권한(roles)·Ads API 프로파일은 공식 문서 기준으로 확인하세요. PII 포함 리포트는 별도 승인이 필요할 수 있습니다.

③ 매주 자동 실행 (스케줄)

이 전체 파이프라인을 매주 월요일 오전 9시에 자동 실행되게 해줘.
(mac/linux면 cron, Windows면 작업 스케줄러로)
# 예: cron (매주 월 09:00)
0 9 * * 1  cd ~/amazon-weekly && ./run_weekly.sh >> reports/run.log 2>&1

완성 후엔 사람이 손대는 시간이 0분 — 월요일 아침 reports/에 새 리포트가 쌓입니다. 슬랙/메일 발송까지 같은 방식으로 추가 가능.
