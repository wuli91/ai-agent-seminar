기업용 AI 에이전트 서비스·플랫폼 비교

AI3(웍스AI)와 유사한 국내·해외 기업용 AI 에이전트 서비스, 그리고 AWS Bedrock·Google Vertex AI(+Azure) 클라우드 플랫폼을 한 문서에 비교했습니다.

먼저 — 3개 레이어로 보면 쉽다 ① 모델 레이어(Claude·GPT·Gemini·EXAONE 등 LLM 자체) ② 클라우드 플랫폼 레이어(Bedrock·Vertex·Azure — 모델을 골라 에이전트를 만들고 운영) ③ 애플리케이션·서비스 레이어(완성된 업무 비서/에이전트 — AI3·뤼튼·Glean·Copilot 등). 같은 "AI 에이전트"라도 어느 층을 사는지가 다릅니다.
⚠ 빠르게 변하는 영역제품명·가격·밸류에이션·출시단계(발표 vs GA)가 분기마다 바뀝니다. 가격은 대부분 비공개·맞춤이며 성능/절감 수치는 벤더 주장이 많습니다. 도입 전 재확인·PoC 필수.

3. AWS Bedrock vs Google Vertex AI (vs Azure AI Foundry)

"완성 서비스"가 아니라 직접 에이전트를 만들고 운영하는 클라우드 플랫폼입니다. 셋 다 멀티모델·관리형 런타임·가드레일·소비기반 과금·"고객 데이터 모델 학습 미사용"을 제공합니다.

선택 기준

· AWS Bedrock: 이미 AWS 네이티브(IAM·VPC) + 멀티모델 + 프레임워크 비종속 프로덕션 런타임(AgentCore)이 필요할 때. Claude를 AWS 경계·결제 안에서.

· Google Vertex AI: GCP·BigQuery 데이터 스택, Gemini 가성비·장컨텍스트, 로우코드+코드 일관 + 지식근로자 앱 통합.

· Azure AI Foundry: M365/Azure 중심 + GPT와 Claude를 한 곳에서.

· Anthropic API 직접: 멀티클라우드 중립 + 최신 Claude를 가장 먼저·라우팅 지연 없이.

Claude 사용 기업(=현재 우리)에게 핵심Claude는 Bedrock·Vertex·Azure·Anthropic 직접 네 경로 모두에서 제공됩니다. 즉 모델이 클라우드 선택을 강제하지 않습니다. 결정은 기존 클라우드 정합성·거버넌스·결제 통합·런타임 기능으로 — 이는 멀티클라우드 협상력·이전성에 유리합니다.
