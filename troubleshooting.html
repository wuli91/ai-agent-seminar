막혔을 때 — 트러블슈팅 가이드

터미널이 처음이어도 괜찮습니다. 막히는 건 실력 문제가 아니라 통과 의례입니다. 아래 표에서 내 증상을 찾아 클릭하세요. 설치 명령은 사전 준비 가이드와 동일합니다.

먼저 기억할 것 한 가지 에러 메시지는 무섭게 생겼지만, 그냥 "왜 안 되는지에 대한 힌트"입니다. 이해 안 되면 그대로 복사해서 Claude(claude.ai)에 "이거 무슨 뜻이야?"라고 물어보면 대부분 풀립니다.

1. 터미널 기초 사전지식 — 이것만 알면 됩니다

터미널(macOS) / PowerShell(Windows)은 마우스 대신 글자로 컴퓨터에 지시하는 창입니다. 이 가이드에 나오는 건 아래가 전부입니다.

이것뜻예시
cd폴더로 이동한다 (change directory)cd ~/Desktop/실습
~내 홈 폴더 (내 계정의 시작 위치)~/Desktop = 내 바탕화면
pwd지금 내가 어느 폴더에 있는지 보여준다pwd 입력 → 현재 경로 출력
경로 입력, 손으로 안 쳐도 됩니다 폴더를 터미널 창에 마우스로 끌어다 놓으면 경로가 자동으로 입력됩니다. macOS는 터미널 창에, Windows는 PowerShell 창에 드래그하면 됩니다. cd (띄어쓰기 포함)를 먼저 친 뒤 폴더를 드래그하고 Enter — 이게 제일 쉽습니다.
  • 경로에 공백이나 한글이 있으면 따옴표로 감싸세요: cd "~/Desktop/실습 폴더" (드래그하면 자동 처리되는 경우가 많습니다)
  • 위쪽 화살표(↑)를 누르면 방금 친 명령이 다시 나옵니다. 긴 명령을 다시 칠 필요 없습니다.
  • 붙여넣기: macOS는 ⌘+V, Windows PowerShell은 Ctrl+V 또는 마우스 오른쪽 클릭.

2. 설치 문제

2-a. macOS: curl ... | bash 가 실패한다

증상설치 명령을 붙여넣었는데 Could not resolve host, Connection timed out, SSL certificate problem 같은 빨간 글씨가 나온다.

원인대부분 네트워크 문제입니다 — 인터넷이 끊겼거나, 회사 프록시/방화벽이 외부 다운로드를 막는 경우.

해결순서대로 시도하세요.

  1. 브라우저에서 아무 사이트나 열어 인터넷이 되는지 먼저 확인
  2. 명령을 한 줄 통째로 다시 복사해서 붙여넣기 (중간에 줄바꿈이 끼면 실패합니다)
  3. 다시 시도:
# 사전 준비 가이드와 동일한 설치 명령
curl -fsSL https://claude.com/install.sh | bash

그래도 같은 에러면 회사 네트워크가 막고 있을 가능성이 큽니다 → 4. 사내 네트워크 참고. 휴대폰 핫스팟으로 잠깐 바꿔서 되면 확실히 네트워크 문제이니, IT/진행자에게 알려주세요.

2-b. Windows: "이 시스템에서 스크립트를 실행할 수 없으므로…" (스크립트 실행 비활성화)

증상irm https://claude.com/install.ps1 | iex 를 넣으면 "스크립트 실행이 사용 안 함으로 설정되어 있으므로 로드할 수 없습니다" 비슷한 빨간 메시지가 나온다.

원인Windows의 보안 설정(ExecutionPolicy)이 스크립트 실행을 막아 둔 상태입니다. 회사 노트북이면 회사가 일부러 잠가둔 것일 수 있습니다.

해결

  1. 회사 노트북이라면 먼저 IT/보안팀에 확인하세요 — 정책으로 잠근 경우 임의로 풀면 안 됩니다.
  2. 개인 노트북이거나 IT가 괜찮다고 하면, PowerShell에서 현재 설정을 확인:
# 현재 정책 확인
Get-ExecutionPolicy

# Restricted 라고 나오면, 내 계정에 한해 완화 (IT 확인 후)
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

변경 후 PowerShell 창을 새로 열고 설치 명령을 다시 실행하세요.

irm https://claude.com/install.ps1 | iex

설정 변경 자체가 막혀 있다면(회사 정책) IT에 요청하는 수밖에 없습니다 — 그대로 세미나에 오셔도 됩니다. 3회차에 함께 해결합니다.

2-c. "관리자 권한이 필요합니다" / Permission denied

증상설치 중 관리자 암호를 묻거나, Permission denied, "이 작업을 수행할 권한이 없습니다" 가 나온다.

원인회사 노트북은 보통 일반 사용자 계정이라 소프트웨어 설치 권한이 없습니다.

해결이건 명령어로 풀 수 있는 문제가 아닙니다. IT 부서에 요청하세요. 요청할 때 이렇게 말하면 빠릅니다:

IT에 보낼 요청 예시 "업무용 AI 도구 Claude Code를 설치하려고 합니다. 설치 권한 부여 또는 대리 설치를 부탁드립니다. 설치 방법: macOS는 curl -fsSL https://claude.com/install.sh | bash, Windows는 PowerShell에서 irm https://claude.com/install.ps1 | iex 입니다."

세미나 전까지 안 되면 그대로 오세요 — 진행자가 함께 봐 드립니다.

2-d. 설치는 끝났는데 claude: command not found (가장 흔함)

증상설치가 성공한 것 같은데 claude --version 을 치면 command not found(맥) 또는 "용어가 인식되지 않습니다"(윈도우)가 나온다.

원인설치 직후에는 지금 열려 있는 터미널 창이 새 프로그램을 아직 모르는 상태입니다. 십중팔구 이것입니다.

해결

  1. 지금 터미널/PowerShell 창을 완전히 닫고, 새 창을 열어서 다시 시도 — 이걸로 대부분 해결됩니다.
  2. 새 창에서 확인:
claude --version
  1. 그래도 안 되면 노트북을 한 번 재부팅 후 다시 시도.
  2. 여전히 안 되면 "PATH"라는 설정(컴퓨터가 프로그램을 찾는 위치 목록)에 등록이 안 된 경우입니다. 이건 환경마다 달라서, 설치 때 나온 메시지를 그대로 복사해 Claude(claude.ai)에 물어보거나 진행자/IT에 보여주는 게 가장 빠릅니다.

3. 로그인 문제

3-a. claude 를 실행했는데 브라우저가 안 열린다

증상로그인하라는데 브라우저 창이 자동으로 뜨지 않는다.

원인환경에 따라 자동으로 브라우저를 못 여는 경우가 있습니다. 고장이 아닙니다.

해결터미널 화면을 보면 로그인용 주소(URL)가 글자로 표시되어 있습니다. 그 URL을 마우스로 긁어 복사한 뒤, 브라우저 주소창에 직접 붙여넣어 여세요. 로그인을 마치면 터미널로 돌아와 계속 진행됩니다.

3-b. 개인 계정으로 잘못 로그인했다

증상회사 계정 대신 개인 Gmail 등으로 로그인해 버렸다. 회사 워크스페이스가 안 보인다.

원인브라우저에 개인 계정이 이미 로그인되어 있으면 그 계정으로 이어지기 쉽습니다.

해결Claude Code 안에서 로그아웃 후 다시 로그인하세요.

# claude 실행 중이라면 그 안에서 입력
/logout

# 다시 로그인 (이번엔 회사 계정 선택)
/login

브라우저 쪽에서도 claude.ai 에서 개인 계정을 로그아웃해 두면 헷갈리지 않습니다. 개인 계정으로는 업무 데이터를 다루지 마세요 — 회사 계정 초대가 안 왔다면 도입 담당에게 문의.

3-c. 로그인이 무한 반복된다

증상브라우저에서 로그인을 분명히 마쳤는데, 터미널이 계속 로그인하라고 한다. 혹은 매번 다시 로그인을 요구한다.

원인인증 정보가 제대로 저장되지 못한 경우입니다. 브라우저의 강한 추적 차단/시크릿 모드, 혹은 회사 보안 프로그램이 원인일 수 있습니다.

해결순서대로:

  1. 터미널과 브라우저를 모두 닫고, 새 터미널에서 claude 다시 실행
  2. 시크릿/프라이빗 창이 아닌 일반 브라우저 창으로 로그인 (기본 브라우저를 Chrome/Edge로)
  3. /logout → /login 으로 처음부터 다시
  4. 그래도 반복되면 회사 보안 프로그램이 끼어드는 경우일 수 있으니 IT/진행자에게 문의

4. 사내 네트워크 — 프록시 · 방화벽 · SSL 검사

회사 네트워크에서만 안 된다

증상이런 메시지가 나온다 — Connection timed out(연결 시간 초과), certificate 가 들어간 오류(인증서 오류), proxy 관련 오류. 집이나 핫스팟에서는 잘 되는데 회사에서만 안 된다.

원인회사 프록시/방화벽이 외부 연결을 막거나, 보안 장비가 SSL(암호화 통신)을 중간 검사하면서 인증서 오류를 일으키는 경우입니다. 내 잘못이 아니고, 내가 명령어로 풀 수 있는 문제도 아닙니다.

해결IT 부서에 아래처럼 요청하세요.

IT에 보낼 요청 예시 "업무용 AI 도구(Claude) 사용을 위해 claude.ai 와 anthropic.com 도메인(하위 도메인 포함) 허용을 요청합니다. 프록시 환경이라면 적용할 프록시 설정값도 안내 부탁드립니다."

프록시를 쓰는 회사라면 IT가 HTTPS_PROXY 라는 환경변수 값을 알려줄 수 있습니다. 값은 반드시 IT가 안내한 것을 쓰세요 (아래는 형식 예시일 뿐입니다).

# macOS (터미널) — 값은 IT가 알려준 주소로
export HTTPS_PROXY=http://proxy.example.com:8080

# Windows (PowerShell) — 값은 IT가 알려준 주소로
$env:HTTPS_PROXY = "http://proxy.example.com:8080"

이 설정은 그 터미널 창에서만 적용됩니다(창을 닫으면 사라짐). 매번 설정하기 번거롭거나 인증서 오류가 계속되면, 영구 설정 방법까지 포함해 IT에 맡기는 게 안전합니다.

5. 실행 중 문제

5-a. 응답이 멈춘 것 같다

증상Claude가 한참 동안 아무 말이 없다. 멈춘 건지 일하는 건지 모르겠다.

원인여러 파일을 읽거나 큰 작업을 할 때는 원래 시간이 걸립니다. 화면에 작업 중 표시(스피너 등)가 돌고 있으면 일하는 중입니다.

해결

  • 1~2분은 기다려 보세요. 진행 표시가 있으면 정상입니다.
  • 너무 길거나 잘못된 방향 같으면 Esc 키를 누르세요 — 현재 작업이 중단되고, 다시 지시할 수 있습니다. 망가지지 않으니 부담 없이 누르세요.
  • 그래도 먹통이면 터미널 창을 닫고 새로 열어 claude 를 다시 실행하면 됩니다.

5-b. 갑자기 뭔가 물어보는 화면이 떴다 — 승인 화면입니다

증상"이 작업을 실행할까요?" 식의 선택지가 떴는데 눌러도 되는지 모르겠다.

원인고장이 아니라 안전장치입니다. Claude Code는 파일을 만들거나 수정·삭제하기 전에 반드시 먼저 보여주고 허락을 구합니다. 읽기만 할 때는 묻지 않습니다.

해결3가지 습관 (3회차 내용 그대로):

  1. 무엇을 하겠다는 건지 읽어보고 괜찮으면 예(Yes) 선택
  2. 의심되면 거절(No) — 그리고 한국어로 다시 지시하면 됩니다. 거절해도 아무것도 망가지지 않습니다.
  3. 애초에 중요 폴더는 복사본으로 실습하면 마음이 편합니다.

5-c. "사용량 한도" 메시지가 나온다

증상usage limit / 한도 도달 같은 메시지와 함께 잠시 사용이 막힌다.

원인플랜마다 일정 시간 동안 쓸 수 있는 양이 정해져 있고, 그걸 다 쓴 상태입니다. 고장이 아닙니다.

해결보통 시간이 지나면 자동으로 풀립니다(메시지에 언제 풀리는지 표시되는 경우가 많습니다). 업무상 자주 막힌다면 플랜 조정이 필요한 신호이니 도입 담당자에게 문의하세요.

5-d. 실수로 파일을 잘못 바꿨다

증상승인했더니 생각과 다르게 파일이 바뀌었거나 옮겨졌다.

원인지시가 모호했거나 승인 화면을 빠르게 넘긴 경우. 누구나 한 번씩 겪습니다.

해결

  1. 당황하지 말고, 같은 대화에서 Claude에게 한국어로 부탁하세요:
방금 작업 되돌려줘. 어떤 파일을 어떻게 바꿨는지 먼저 보여줘.
  1. Claude가 무엇을 바꿨는지 알고 있으므로 대부분 되돌릴 수 있습니다. 되돌리는 작업도 승인 화면을 읽고 진행하세요.
  2. 이래서 실습 원칙이 "복사본으로 연습, 원본 보존"입니다 — 복사본이었다면 그냥 지우고 다시 복사하면 끝.
  3. 중요한 원본이 걸려 있는데 되돌리기가 확실치 않으면, 거기서 멈추고 진행자에게 보여주세요.

6. 그래도 안 되면 — 3단계 안전망

  1. 에러 메시지를 그대로 복사해서 Claude 앱(claude.ai)에 붙여넣고 "이 에러 무슨 뜻이고 어떻게 고쳐?"라고 물어보세요. 의외로 여기서 대부분 끝납니다.
  2. 그래도 모르겠으면 진행자 또는 도입 담당자에게 문의 — 에러 메시지 화면을 캡처해서 보내면 가장 빠릅니다.
  3. 세미나 중이라면 옆 사람과 페어로 진행하세요. 한 명 화면으로 같이 실습해도 배우는 건 똑같습니다.
