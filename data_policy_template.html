1. 데이터 3등급 — 신호등 기준

등급기준예시

🔴 금지
입력 불가		 유출 시 법적 책임·중대한 손실이 생기는 정보 고객·임직원 개인정보(이름·연락처·주민번호), 미공개 재무·실적, 계약서·법무 문서 원문, 비밀번호·API 키·인증서, 미공개 설계도·소스코드 [회사에 맞게 수정]

2. 🟡 조건부 데이터의 익명화 절차

  1. 식별자 제거: 사람 이름→직무(예: "김OO"→"영업담당"), 회사명→"A사", 금액·날짜는 필요시 비율/상대값으로.
  2. 붙여넣기 전 1회 재확인: "이 내용이 외부 메일에 첨부돼도 괜찮은가?" — 아니라면 더 가린다.
  3. 판단이 안 서면: 입력하지 말고 정책 담당 [담당자]에게 먼저 문의.

3. 계정·도구 규칙

  • 업무 사용은 회사가 발급한 계정(Team 플랜 이상)으로만. 개인 무료 계정에 업무 데이터 입력 금지.
  • 허용 도구 목록: [ Claude 앱 / Claude Code / 기타 승인 도구 ] — 목록 외 도구는 사전 승인 필요.
  • Claude Code 사용 시: 비밀번호·키·인증서가 든 폴더에서 실행 금지, 삭제·이동은 승인 화면 확인 후 진행.

4. 위반·사고 발생 시

  • 금지 데이터를 입력한 사실을 인지하면 즉시 [담당자/보안팀]에 보고합니다. (자진 보고는 불이익 없음 — 숨기는 것이 더 큰 문제)
  • 담당자는 해당 대화 삭제, 영향 평가, 필요시 [개인정보보호 책임자] 보고를 진행합니다.

5. 사용자 셀프 체크 (입력 전 10초)

  • 이 내용, 외부 메일에 첨부해도 문제없는가?
  • 사람 이름·연락처·고객 식별 정보가 들어 있지 않은가?
  • 미공개 숫자(재무·실적·단가)가 들어 있지 않은가?
  • 지금 쓰는 계정이 회사 발급 계정인가?
  • 판단이 애매한가? → 애매하면 넣지 않는다.
