진행자 가이드

AI Agent 활용 세미나를 처음 진행하는 사내 진행자를 위한 운영 매뉴얼입니다. 사전 준비 → 세션별 분 단위 진행안·데모 대본 → 예상 Q&A → 트러블 대응 → 사후 관리까지, 이 한 장으로 진행할 수 있습니다.

1. 개요 — 이 가이드와 세미나의 구조

이 세미나는 D-level(디렉터·임원)과 비개발 직군이 각자 자기 업무에 Claude를 실제로 도입하게 만드는 과정입니다. 진행자의 역할은 "강의"가 아니라 참석자가 직접 손으로 해보게 만드는 것입니다. 설명이 길어지면 실습 시간을 깎아 먹습니다 — 시간이 모자라면 항상 설명을 줄이고 실습을 지키세요.

구성내용비고
세션 수4세션 × 약 90분기본: 하루 3시간(2세션) × 2일. 심화 시 하루 1세션으로 3~4일
회당 구성개념·데모 45분 → 핸즈온 35분 → 정리·과제 10분모든 세션 공통 골격
도구Session 1~2: Claude 앱 / Session 3~4: Claude Code참석자는 회사 발급 계정(Team 이상 권장) 사용
세션별 산출물S1 프롬프트 감각 → S2 직무 프로젝트 1개 → S3 자동화 1건 → S4 30/60/90일 계획"끝나면 손에 남는 것"을 매 세션 강조
진행자가 외울 한 문장 (세션별 클로징 멘트) S1 "AI는 똑똑한 인턴, 검토는 내 몫" · S2 "설명은 한 번, 결과물은 매번" · S3 "명령어 대신 한국어, 실행 대신 승인" · S4 "리더가 직접 쓰는 조직이 빨라진다"

2. 사전 준비 체크리스트

D-7 (일주일 전)

  • 플랜·계정: 회사 계정(좌석) 발급 완료 확인 — 참석자 전원이 회사 발급 계정으로 claude.ai 로그인 가능한지 명단 대조 (요금제 & 보안 비교 참고, Team 이상 권장)
  • 실습 자료: workshop_samples 폴더를 ZIP으로 압축해 배포 채널 준비 (사내 메신저/공유 드라이브) — 내용물은 실습 소재 모음 ④ 표와 일치하는지 확인
  • 데이터 정책: "허용/조건부/금지" 데이터 기준 1장을 보안팀과 확정해 세미나 전에 전사/참석자 공지 — 데이터 정책 1장 템플릿의 노란 칸만 채우면 됨
  • 참석자 안내 메일 발송: 사전 준비 가이드 링크 + "반복 업무 1~2가지 적어오기" 과제 + Session 3 전까지 Claude Code 설치 안내

D-1 (전날)

  • 회의실: 좌석·전원(멀티탭)·프로젝터/대형 화면 연결 테스트 — 터미널 데모는 글자가 작으므로 화면 확대(폰트 크게) 미리 설정
  • 와이파이: 게스트/사내망에서 claude.ai 접속과 Claude Code 동작이 막히지 않는지 실제 확인
  • 데모 리허설: 아래 세션별 데모 대본 4개를 본인 노트북으로 처음부터 끝까지 1회 실행 — 특히 S3의 workshop_samples CSV 데모는 폴더 경로까지 그대로 재현
  • 슬라이드: index.html → session_01~04.html이 발표 PC에서 열리고 ←/→ 키로 넘어가는지 확인
  • 화면 캡처: AI 앱 기능 지도용 실제 캡처를 당일 UI 기준으로 갱신 — assets/captures/README.md의 촬영 목록(7장)대로 찍어 그 폴더에 넣으면 문서에 자동 표시됨. Q&A "ChatGPT/Gemini/웍스AI랑 뭐가 달라요?" 대응용

당일 (시작 30분 전)

  • 체크인: 입장하는 참석자마다 "claude.ai 로그인 되셨어요?" 한 명씩 확인 — 안 된 사람은 자리 표시해 두기
  • 지원 인력: 미설치·미로그인자 지원 담당 1명(보조 진행자)을 배치 — 본 진행과 분리해야 전체 진행이 안 밀림
  • 실습 ZIP 배포 링크를 화면/칠판에 게시
  • 데이터 정책 1장을 인쇄해 게시하거나 첫 슬라이드 전에 한 번 더 공지

3. 세션별 진행안

Session 1 — AI Agent 기초 (90분 · 슬라이드 14장)

핵심 메시지: "AI는 똑똑한 인턴 — 빠르지만 결과 검토는 사람 몫"이라는 감각과, "외부 메일에 첨부할 수 있나?"라는 데이터 투입 기준 한 가지를 심는 세션입니다.

시간슬라이드진행 내용
0–5분#1–2 커버·오늘의 목표환영 인사, "코딩 몰라도 됩니다" 안심시키기
5–10분#3 전체 여정4세션 지도 보여주기 — 앱(S2)과 Code(S3) 두 도구 예고
10–20분#4–5 LLM이란·에이전트란"초거대 자동완성"과 환각 개념, 챗봇 vs 에이전트 그림 설명
20–27분#6 두 도구 비교앱=대화형, Code=내 컴퓨터 파일. "대부분 앱으로 시작 → 반복은 Code"
27–37분#7–8 보안·거버넌스가장 천천히. 넣어도 되는 것/안 되는 것, 회사 데이터 정책 1장과 연결
37–45분#10–11 프롬프트 6원칙 + Before/After라이브 데모 (아래 대본) — 6원칙은 데모 후 정리로
45–80분#9 핸즈온로그인 → 내 업무 질문 → 프롬프트 개선. 진행자는 순회
80–90분#12–14 실수·과제·마무리자주 하는 실수 4개 빠르게, 과제 공지, 한 줄 클로징

데모 대본 — 회의록 요약 Before/After (약 8분)

  1. 실습 소재 모음 ①의 샘플 회의록(5/12 신제품 TF 회의)을 화면에 띄우고 전체 복사한다. "일부러 정리 안 된 진짜 회의 메모 같은 자료입니다."
  2. Before: claude.ai 새 대화에 회의록을 붙여넣고 그냥 회의록 요약해줘 — 결과를 보여주며 "무난하죠? 그런데 임원 보고에 그대로 못 씁니다."
  3. After: 새로 요청 — 이 회의록을 정리해줘. 결정사항/담당자/기한은 표로, 미해결 이슈는 따로 불릿으로, 임원 보고용 한 페이지 분량 — 표가 나오면 "차이는 '무엇을, 누구에게, 어떤 형식으로'를 말해준 것뿐입니다."
  4. 한 방: 이어서 빠뜨린 위험 요소가 있는지도 짚어줘 — 회의록 끝의 "인증 재취득 비용 아무도 확인 안 함" 함정을 AI가 짚어내면 박수 포인트. 동시에 "이런 것도 사람이 맞는지 확인해야 합니다"로 검토 원칙 복습.

실습 순회 포인트

  • "뭘 물어볼지 모르겠어요" — 가장 흔함. 적어온 반복 업무를 묻거나, 샘플 회의록으로 데모 따라 하기를 권유.
  • 첫 답이 평범해서 실망 — "6원칙 중 형식(④)을 추가해 보세요"라고 한 가지만 콕 집어 개선시키기. 개선 전후 비교가 이 세션의 학습 그 자체.
  • 실제 기밀을 붙여넣으려는 참석자 — 즉시, 그러나 부드럽게: "이름·숫자만 가리고 다시 해보시죠." 전체에 공유할 만하면 한 번 더 전체 공지.

시간 부족 시 생략: #3 전체 여정(말로 30초 대체), #8 거버넌스(D-level 비중이 낮은 회차일 때). #7 보안은 절대 생략 금지.

Session 2 — Claude로 일하기, Cowork (90분 · 슬라이드 13장)

핵심 메시지: 한 번 쓰고 마는 대화를 Projects로 반복 가능한 업무 도구로 바꾼다 — "설명은 한 번, 결과물은 매번".

시간슬라이드진행 내용
0–7분#1–2 커버·복습지난 과제("AI로 시도해본 일") 1~2명 가볍게 발표시키기
7–13분#3 Cowork 개념맥락 기억·자료 읽기·결과물 생성 3박자
13–23분#4 Projects지침+문서+대화 누적 그림. "매번 배경 설명 다시 하기의 종말"
23–30분#5 Artifacts"문서로 만들어줘/표로 정리해줘"라고 하면 옆에 결과물 생성
30–36분#6 직무별 표참석자 직무를 호명하며 해당 줄 읽기 — 핸드아웃 예고
36–45분#7–8 데모 ①②라이브 데모 (아래 대본): Project 생성 + PDF 요약
45–80분#9 핸즈온내 직무 프로젝트 만들기 + Artifact 1개. 순회
80–90분#10–13 습관·한계·과제·마무리"앱만으로 아쉬운 순간" → Session 3 예고로 연결

데모 대본 — Project 생성 + PDF 요약 (약 9분)

  1. claude.ai에서 새 Project 생성 — 이름은 참석자 다수 직무에 맞춰 (예: "주간보고 작업실"). "이름은 업무 단위로 짓는 게 요령입니다."
  2. 프로젝트 지침에 실습 소재 모음 ②의 지침 템플릿을 붙여넣고 [역할]/[톤과 형식] 두 칸만 그 자리에서 채운다 — "이 한 번이 앞으로 모든 대화에 자동 적용됩니다."
  3. 참고 자료로 공개 PDF 보고서 1개 업로드 (리허설 때 쓴 공개 자료. 마땅치 않으면 샘플 회의록 파일 common/sample_meeting_notes.md로 대체).
  4. session_02 데모 ① 프롬프트를 그대로 입력: 
    이 문서를 임원 보고용으로 정리해줘.
- 핵심 결론 3가지 (각 한 줄)
- 의사결정이 필요한 항목 표 (이슈/선택지/추천)
- 리스크와 다음 단계
- 전체 한 페이지 이내
  5. 결과가 Artifact로 뜨면 더 짧게, 표 중심으로라고 한 번 다듬어 버전이 쌓이는 것을 보여준다. 마무리 멘트: "20장이 한 페이지로. 단, 숫자는 원문과 대조하세요."

실습 순회 포인트

  • 지침 칸을 비워두고 그냥 대화부터 시작 — 템플릿의 [역할]과 [톤과 형식]만이라도 채우게. 이게 이 세션의 핵심 산출물.
  • "올릴 자료가 없어요" — 실습 소재 모음의 샘플 회의록을 안내. 실제 자료를 쓰려는 경우 익명화 여부 확인.
  • Artifact가 안 생기고 채팅 답변만 나옴 — "문서로 만들어줘 / 표로 정리해줘"처럼 결과물 형태를 명시하도록 코칭 (슬라이드 #5의 사용법 그대로).
  • 한 번 만들고 끝내려는 참석자 — "더 짧게/톤 바꿔/표로" 2~3회 다듬기까지가 실습 완료 조건임을 상기.

시간 부족 시 생략: #6 직무별 표(핸드아웃으로 대체 안내), #8 데모②(초안 작성 — 데모①만으로 충분). #4 Projects와 핸즈온은 생략 불가.

Session 3 — Claude Code 입문 (90분 · 슬라이드 12장)

핵심 메시지: 터미널 공포 제거 — "명령어 대신 한국어, 실행 대신 승인". 읽기는 자유, 바꾸기는 내 승인 후라는 안전 구조를 몸으로 익히게 합니다.

시간슬라이드진행 내용
0–5분#1 커버"검은 화면 겁내지 마세요" — 오늘은 복붙과 한국어만
5–13분#2 앱과의 차이폴더·수십 개 파일·반복 자동화 — 지난주 "번거로운 일" 과제 회수
13–20분#3–4 터미널 3가지 + 첫 실행열기 → claude → 한국어 지시. 맥은 폴더 드래그 팁
20–28분#5 승인 모드천천히. 읽기 자유/바꾸기 승인, 복사본·승인 읽기·거절 3습관
28–35분#6 비개발자 활용 6종참석자 업무와 연결되는 카드 위주로
35–45분#7 데모라이브 데모 (아래 대본): workshop_samples CSV → 리포트
45–80분#8 핸즈온샘플 폴더에서 자동화 1건. 이 세션이 순회 난이도 최고 — 보조 진행자 필수
80–90분#9–12 잘 쓰는 법·보안·과제·마무리"반복 업무 1가지 골라두기" 과제 강조 (S4 재료)

데모 대본 — workshop_samples에서 CSV → 리포트 (약 10분, 명령 대사 포함)

  1. 터미널을 열고 화면 폰트를 키운 상태에서, 미리 바탕화면에 압축 해제해 둔 샘플 폴더로 이동: 
    # 맥 (폴더를 터미널 창에 끌어다 놓아도 됨)
cd ~/Desktop/workshop_samples
claude
    "여기까지가 외울 것의 전부입니다. 이제부터는 한국어입니다."
  2. 첫 지시 (그대로 타이핑): 이 폴더에 뭐가 있는지 알려줘 — Claude가 폴더 구조를 정리해 주면 "파일을 직접 읽는다는 게 이런 뜻입니다."
  3. 본 작업: amazon_na_sales_sample_level1.csv를 읽고 카테고리별 매출 요약을 마크다운 리포트로 만들어줘. 파일을 만들기 전에 어떤 내용으로 만들지 먼저 보여줘
  4. Claude가 계획/파일 생성 승인 화면을 띄우면 일부러 멈추고: "이 화면이 안전장치입니다. 읽고, 예 또는 아니오. 승인 없이는 아무것도 안 바뀝니다." → 승인.
  5. 생성된 리포트 파일을 열어 보여주고 비교: "손으로 30분 걸릴 일이 30초. 목표 결과물 예시는 index의 Amazon 주간 리포트 예시와 같습니다."
  6. (여유 시) 이어서 방금 리포트에 매출 상위 3개 제품 표도 추가해줘 — "대화로 이어 고친다"를 시연.

실습 순회 포인트

  • 폴더 이동(cd)에서 막힘 — 가장 흔한 정체 구간. 맥은 폴더를 터미널로 드래그, 윈도우는 탐색기 주소창 복사를 알려주거나 보조 진행자에게 인계.
  • claude 명령이 안 됨(미설치) — 본 진행을 멈추지 말고 보조 진행자가 설치 지원 또는 옆 사람과 페어로 묶기 (6장 운영 팁 참조).
  • 승인 화면을 안 읽고 전부 예 — 순회하며 "방금 뭘 승인하셨어요?"라고 물어 한 번 멈추게 하기. 안전 습관이 이 세션의 목표.
  • 에러 메시지에 얼어붙음 — 에러를 복사해 이거 무슨 뜻이야?라고 Claude에게 그대로 묻게 하기 (슬라이드 #8 팁 그대로).
  • 시킬 일이 없다는 참석자 — 샘플 폴더의 과제 문서 활용: hr_recruitment 폴더의 과제 문서를 읽고, 시키는 대로 후보자 비교표를 만들어줘

시간 부족 시 생략: #6 활용 6종(데모로 갈음), #9 잘 쓰는 법(순회 중 개별 코칭으로 흡수). #5 승인 모드와 #10 보안은 생략 금지.

Session 4 — 내 업무에 심기 (90분 · 슬라이드 13장)

핵심 메시지: 잘 된 지시를 자산으로 — CLAUDE.md·슬래시 커맨드로 표준화하고, 리더로서 30/60/90일 도입 계획을 오늘 들고 나가게 합니다.

시간슬라이드진행 내용
0–6분#1–2 커버·목표표준화/확장/결정 — "오늘은 들고 나가는 게 있는 날"

데모 대본 — 실행계획 프롬프트 (약 6분)

  1. 진행자 본인 부서를 예로 슬라이드 #10의 프롬프트를 채워 입력한다: 
    나는 [직무/부서]의 리더야. AI 도입 실행계획을 같이 만들어줘.
- 내 부서가 매주 반복하는 업무 3가지: [적기]
- 각각을 Claude 앱/Claude Code 중 무엇으로, 어떻게 처리할지
- 30/60/90일 계획 표로
- 보안상 주의할 점도 포함
    [적기]에는 실제 업무 3가지를 그 자리에서 채운다 (안 떠오르는 참석자용으로 실습 소재 모음 ③의 직무별 예시를 화면에 잠깐 띄워준다).
  2. 결과 표가 나오면: "초안일 뿐입니다. 담당과 일정은 여러분이 현실에 맞게 고치는 게 두 번째 단계입니다."
  3. 마지막으로 본인 계획에서 "이번 주 시작할 1가지"에 동그라미 치는 모습을 보여준다 — "여러분도 공유 때 이 1가지를 말하게 됩니다."

실습 순회 포인트

  • 반복 업무 3가지를 못 적음 — 실습 소재 모음 ③ 표에서 자기 직무 줄을 그대로 가져가게.
  • 계획이 너무 거창함("전사 시스템 연동") — "이번 주 시작할 1가지"로 끌어내리기. 작게 시작이 원칙.
  • 앱/Code 배정을 헷갈림 — 기준 한 줄: 글·요약·기획은 앱, 여러 파일·반복 처리는 Code (S1 #6 표).
  • 공유를 꺼리는 분위기 — 진행자가 먼저 1분 시범. 직급 높은 분께 미리 첫 타자를 부탁해 두면 잘 풀림.

시간 부족 시 생략: #7 플러그인(슬라이드 #6 끝에 한 줄로 언급), #6 스킬/MCP 축약. #9 30/60/90과 #11 공유는 생략 금지 — 이 세션의 존재 이유.

4. 예상 질문 Q&A — 권장 답변 요지

받아치기보다 슬라이드와 배포 문서로 돌려보내는 것이 안전합니다. 슬라이드에 없는 제품 기능을 즉석에서 약속하지 마세요.

질문권장 답변 요지
"우리 데이터로 AI가 학습하나요?" (보안)회사가 발급한 Team 플랜 이상의 비즈니스 플랜은 입력 데이터를 기본적으로 모델 학습에 사용하지 않습니다. 그래서 개인 계정이 아닌 회사 계정을 쓰는 것. 상세 비교는 요금제 & 보안 비교 문서로 안내.
"틀린 답을 하면 어떡하죠?" (환각)실제로 틀립니다 — 모르면서 그럴듯하게 지어내는 게 환각이고, 틀려도 자신 있게 말합니다. 그래서 원칙이 "AI는 똑똑한 인턴, 검토는 내 몫". 숫자·사실·고유명사는 반드시 원본 대조, 불확실한 건 [확인 필요] 표시를 습관화.
"비용은 얼마나 드나요?"플랜별 가격·조건은 요금제 & 보안 비교, 조직 단위 비용 구조는 TCO 비교 문서 참조. 도입 관점에서는 "절감 시간 측정(ROI)"이 핵심이며 30/60/90일 계획에 포함돼 있음. 구체적 계약 조건은 도입 담당에게.
"내 일자리가 없어지는 건가요?"이 과정의 관점: AI는 초안·요약·정리의 0→70%를 빠르게 하고, 판단·검토·책임은 사람이 채웁니다. 줄어드는 건 일자리가 아니라 반복 작업 시간이고, 격차는 "쓰는 사람과 안 쓰는 사람" 사이에서 벌어집니다 — 그래서 오늘 직접 배우는 것.
"ChatGPT랑 뭐가 다른가요?"오늘 배우는 핵심(프롬프트 6원칙, 검토 습관, 데이터 기준)은 어떤 AI 도구를 쓰든 동일하게 통합니다. 다만 이 세미나는 회사가 발급·관리하는 Claude 계정(데이터 학습 미사용, 중앙관리)과, 내 컴퓨터 파일까지 다루는 Claude Code까지를 묶어 실습합니다. 타 도구와의 상세 비교는 AI 에이전트 지형도 등 심화 자료로 안내.
"터미널은 무서워요 / 개발자나 쓰는 것 아닌가요?"오늘 외울 것은 claude 한 단어뿐, 나머지는 전부 한국어로 부탁합니다. 그리고 파일을 바꾸기 전엔 항상 승인 화면이 떠서 내가 "예"를 해야 실행됩니다 — "명령어 대신 한국어, 실행 대신 승인".
"이거 회사 정책상 써도 되는 건가요?"회사가 계정을 발급한 도구·범위 안에서는 됩니다. 무엇을 넣어도 되는지는 공지된 데이터 정책 1장이 기준이며, 보안팀 가이드가 항상 우선. 애매하면 익명화하거나 넣지 않는 것이 기본값.
"한국어 잘 되나요?"오늘 모든 데모와 실습이 한국어로 진행됩니다 — 방금 보신 회의록 정리/CSV 리포트가 그 답입니다. 톤(보고체/안내체)도 프롬프트로 지정할 수 있고, 어색하면 "더 자연스럽게"로 다듬으면 됩니다.
"사용량 제한이 있나요? 많이 쓰면 끊기나요?"플랜별로 사용량 정책이 다르므로 요금제 & 보안 비교에서 확인. 좌석·사용량 관리는 회사 계정 관리자(도입 담당)가 하므로, 실습 중 한도 관련 메시지가 보이면 진행자에게 알려달라고 안내.
"결과를 어디까지 믿어도 되나요?"기준선: 초안·요약·정리·아이디어는 믿고 시키되, 사실·숫자·인용·법적 판단은 검증 후 사용. 외부로 나가는 산출물은 반드시 사람이 최종 검토 — "사람이 최종 검토"는 도입 거버넌스 원칙이기도 합니다.
"기밀 회의록도 정리시켜도 되나요?"기준은 "외부에 보내도 되는 메일에 첨부할 수 있나?". 안 되면 이름·숫자를 가린 익명화 사본으로 하거나 넣지 않습니다. 데이터 정책 1장의 등급(허용/조건부/금지) 확인이 먼저.
"Claude Code가 제 파일을 지우면 어쩌죠?"파일을 만들거나 수정·삭제하기 전에 반드시 승인을 묻고, 거절(No)하면 실행되지 않습니다. 거기에 더해 실습 3습관 — 복사본으로 연습, 승인 화면 읽기, 의심되면 거절 — 을 지키면 안전합니다.

5. 트러블 대응 — 실습이 막혔을 때

원칙: 전체 진행을 멈추지 않는다 한 명의 기술 문제로 전체 35분 핸즈온을 세우지 마세요. 개별 문제는 보조 진행자에게 인계하고, 같은 문제가 3명 이상이면 그때만 전체 공지로 해결합니다.
  1. 기술 문제(설치·로그인·에러)는 트러블슈팅 문서(증상별 해결 절차)로 안내합니다 — 참석자 스스로 따라 할 수 있게 링크를 실습 ZIP과 함께 배포해 두세요.
  2. 그래도 진행 불가(미설치, 계정 미발급, 노트북 문제)이면 옆 사람과 페어로 묶기 — 한 화면에서 한 명이 타이핑, 한 명이 프롬프트를 부릅니다. 페어가 오히려 학습 효과가 좋다고 안내하면 분위기가 살아납니다.
  3. 에러 메시지는 일단 Claude에게: 메시지를 복사해 "이거 무슨 뜻이야?"라고 묻게 하는 것 자체가 이 세미나가 가르치는 사용법입니다.
  4. 와이파이/서비스 장애로 실습 전체가 불가하면: 데모를 진행자 화면(테더링)으로 이어가고, 핸즈온은 "프롬프트를 종이/메모장에 완성해 두기"로 전환 → 복구 후 또는 과제로 실행.
  5. 시간이 밀리면 각 세션의 "생략 가능 슬라이드"부터 자릅니다. 실습과 보안 슬라이드는 마지막까지 지킵니다.
