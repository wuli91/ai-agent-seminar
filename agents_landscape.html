A. Claude 외 자율형 에이전트 — 무엇이 다른가

Claude Code는 의도적으로 단일 스레드 마스터 루프(사람 승인 중심·투명성·신뢰성 우선)입니다. 대부분의 타 프레임워크는 멀티에이전트 오케스트레이션(역할/그래프/대화 기반)이거나, 더 높은 자율성(장시간 백그라운드 실행)을 지향합니다.

근본 차이 4가지

· 자율성: Claude=승인 중심 / Devin·Manus·OpenClaw=장기 백그라운드 자율

· 실행 위치: Claude=로컬 / Devin·Manus·Bedrock=클라우드 / OpenHands·Hermes=온프레

· 모델 종속: Claude=Claude 전용 / 오픈진영=LLM 교체 가능

· 개방성: 오픈소스는 추론체인까지 감사·커스터마이즈(데이터 주권에 유리)

응용 기대

· 역할 기반 멀티에이전트 팀(기획·개발·검수 분담)

· 목표만 주면 장기 자율 태스크(리서치·코딩·파일/메신저 자동화)

· 셀프호스트·온프레로 데이터 주권·모델 자유(규제 산업)

· 업계는 MCP·A2A 프로토콜로 상호연결 수렴 중

현실적 한계 (D-level 핵심) 복잡도가 오르면 실패율 70~90% 보고, SWE-bench 실측 15~30%. Gartner는 agentic-AI 프로젝트의 40%+가 2027년까지 취소 전망. PoC 비용을 CIO가 최대 1,000% 과소추정한 사례도. 사람 감독(human-in-the-loop)이 여전히 필수 — Claude Code의 "승인 우선" 설계가 오히려 안전 강점입니다.

참고: "open clew" · "hermes agent" 추정

· open clew → OpenClaw(확신도 높음): 2025–2026 바이럴된 오픈소스 셀프호스트 개인 자율 에이전트. 로컬 실행, 메신저(WhatsApp·Telegram 등)가 UI, 100+ 스킬로 셸·파일·웹 자동화. 단, RCE 취약점·악성 확장 등 보안 이슈도 부각.

· hermes agent → Nous Research: ① Hermes 4=오픈웨이트 모델 패밀리, ② Hermes Agent=셀프-임프루빙 오픈소스 에이전트 프레임워크(성공 작업을 재사용 스킬로 축적). 맥락상 ②일 가능성. (두 산출물이 이름 공유 — 혼동 주의)

B. 컴퓨터(화면·마우스·키보드)를 직접 조작하는 에이전트

결론: 예, 존재합니다. 화면(스크린샷)을 보고 마우스·키보드를 조작해 사람 대신 작업하는 "Computer Use / GUI 에이전트"가 2025년 GA/프리뷰로 출시됐습니다. 다만 대부분 브라우저 중심이며, 전체 데스크톱 OS 조작은 아직 신뢰성이 낮습니다.

위험·한계 (보안·신뢰성)

· 프롬프트 인젝션: 웹/문서에 숨긴 악성 지시로 탈취(완화 후도 공격성공 10%대, 0% 아님)

· 정확도: 최상위도 OSWorld 80%대=5건 중 1건 실패. 데스크톱 단계작업일수록 급락

· 속도·비용: 단계마다 스크린샷+추론(작업당 수분)

· 권한·자격증명 위임 위험 → allow-list·자격증명 격리·민감작업 확인·가상환경 필수

엔터프라이즈 권고회계·금융 등 고위험 무인 자동화는 시기상조. 감사 로그·승인 게이트·범위 제한 하에 파일럿부터. 거버넌스는 Copilot Studio(Windows 365 격리)·Claude/OpenAI 엔터프라이즈가 빠르게 갖춰지는 중.

병행 — 장점

· 벤더 락인 회피(가격·정책·가용성 리스크 분산)

· 태스크별 라우팅: 까다로운 수정·리뷰=Claude, 장기 자율=Codex, 대규모 일괄·멀티모달=Gemini

· 이중화(장애·rate limit 시 대체)

· 가격 협상력 + 교차 벤치로 내부 품질기준

· MCP 업계 표준화로 도구·컨텍스트 이식성↑

병행 — 단점/주의 (보안·거버넌스)

· 데이터 노출면 3배: 소스코드가 3개 벤더로 → DPA·학습제외·보존정책 벤더별 개별 계약(HW 설계 IP 특히 민감)

· 거버넌스 파편화(SSO·감사·시크릿스캔·실행권한을 3개 콘솔에서)

· 비용 중복·가시성 저하, 컨텍스트 스위칭 부담

· 자율 에이전트가 각각 코드 실행/PR → 샌드박스·승인 게이트 필수

권장 운영 방식

① Claude Code = 1차 표준(데일리 코딩·리뷰·신뢰성, 사내 자산 보유) ② Codex = 2차(장기 자율·ChatGPT 번들·이중화) ③ Gemini = 한정 파일럿(1M 컨텍스트·멀티모달 필요 팀) ④ 공통 거버넌스 레이어 선구축(엔터프라이즈/Zero-retention 계약·SSO·감사·시크릿스캔·샌드박스·통합 비용 대시보드) ⑤ MCP 표준화로 사내 도구 한 번 구축해 3개 클라이언트 재사용 ⑥ 내부 골든 태스크셋으로 분기별 재평가.
